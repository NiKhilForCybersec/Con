Windows Logs

Windows Event Logs for security monitoring

Windows Event Log Architecture

Windows logging is organized into channels, with Security being the most critical for SIEM:

Log Channel What It Contains SIEM Priority
Security Authentication, authorization, audit events Critical
System OS events, service starts/stops, errors Important
Application Application-specific events Selective
PowerShell Script execution, commands Critical
Sysmon Enhanced process/network/file monitoring Critical

Audit Policy Configuration

Windows doesn't log security events by default. You must enable auditing:

‚ö†Ô∏è Critical Configuration

Enable these audit policies via GPO for security visibility:

  • Logon/Logoff: Success + Failure
  • Account Logon: Success + Failure
  • Process Creation: Success (+ command line logging)
  • Account Management: Success + Failure
  • Object Access: Selective (high volume)

üíº Interview Tip

Question: "What Windows events would you collect for detecting lateral movement?"

Answer: "For lateral movement, I focus on 4624 with Logon Type 3 (network) and Type 10 (RDP), 4648 for explicit credential use like runas, 4688 for process creation with command line to catch PsExec and similar tools, and 4769 for Kerberos service tickets to detect Kerberoasting. I also look at 5140/5145 for share access. On the network side, firewall logs showing SMB (445) and RDP (3389) between workstations are strong indicators."
