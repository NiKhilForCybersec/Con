Application Logs

Web applications, SaaS, and custom application logging

Application Logging for Security

Application logs are often overlooked but critical for detecting application-layer attacks and business logic abuse.

Web Application Logs

Log Type Contents Security Value
Access Logs HTTP requests (URL, status, IP) Attack patterns, scanning, WAF bypass
Error Logs Application errors, stack traces Exploitation attempts, injection
Authentication Logs Login success/failure Credential attacks, account takeover
Audit Logs User actions, data access Insider threat, data exfiltration

SaaS Application Logs

Critical SaaS sources for security monitoring:

  • Microsoft 365: Exchange, SharePoint, Teams activity
  • Google Workspace: Gmail, Drive, Admin console
  • Salesforce: Login history, Setup Audit Trail
  • GitHub/GitLab: Code access, repo changes
  • Slack: Message activity, file sharing

  • MailItemsAccessed - Email accessed
  • FileDownloaded - SharePoint/OneDrive downloads
  • Set-InboxRule - Email forwarding rules
  • Add-MailboxPermission - Mailbox delegation

Custom Application Logging

When building or advising on custom applications, recommend:

  • Structured logging (JSON format)
  • Consistent timestamp format (ISO 8601)
  • User identification in every event
  • Session/request correlation IDs
  • Security-relevant events: auth, access, errors, admin actions
üíº Interview Tip

Question: "How would you detect a business email compromise (BEC)?"

Answer: "BEC indicators in O365 logs: new inbox rules forwarding to external addresses, mailbox permission changes granting access to other users, login from unusual locations or IPs, and mass email access (MailItemsAccessed). I correlate with identity logs for impossible travel and new device sign-ins. Also watch for replies to messages that were auto-forwarded - suggests attacker monitoring conversations."
