Each Layer Explained

1. Collection Layer

This is where data enters the SIEM. Different sources require different collection methods:

Collection Method How It Works Common Use
Agent/Forwarder Software installed on the source that sends logs to SIEM Windows/Linux servers, endpoints
Syslog Standard protocol (UDP/TCP 514) - source pushes logs Network devices, firewalls, Linux
API Pull SIEM queries source's API on a schedule Cloud services (AWS, Azure, SaaS apps)
API Push (Webhook) Source sends to SIEM's HTTP endpoint Cloud-native apps, custom integrations
Database Query SIEM queries database tables directly Legacy applications, CMDB
File Monitoring SIEM watches log files for changes Applications that write to local files
ğŸ’¡ Key Concept: Push vs Pull

Push: The source sends data to the SIEM (syslog, agents, webhooks). More real-time, but SIEM must be reachable.

Pull: The SIEM fetches data from the source (API polling). Works through firewalls, but adds latency.

2. Processing Layer

Raw logs are useless until they're processed. This layer transforms them into structured, searchable data.

Normalization

Different sources use different field names for the same concept. Normalization maps them to a common schema:

Source Original Field Normalized Field
Windows IpAddress src_ip
Linux SRC
Firewall source_address
Windows TargetUserName user
Linux USER
Cloud userPrincipalName

Without normalization, you'd need different queries for every log source. With normalization, one query like src_ip = "192.168.1.100" searches across all sources.

Interview Preparation

ğŸ’¼ Interview Tip

Common Question: "Walk me through SIEM architecture."

Good Answer: "SIEM has five main layers. First, Collection - getting logs from sources using agents, syslog, or APIs. Second, Processing - parsing raw logs into fields, normalizing to a common schema, and enriching with context like GeoIP. Third, Storage - indexed database with hot/warm/cold tiers balancing speed and cost. Fourth, Analysis - correlation engine for detection rules, search interface for hunting, and dashboards for visibility. Fifth, Output - alerts, reports, and automated response through SOAR."

ğŸ’¼ Interview Tip

Common Question: "What's the difference between on-prem and cloud SIEM?"

Good Answer: "On-prem gives you full control over infrastructure and data location, but you manage hardware, scaling, and upgrades. Cloud SIEM removes that burden - the vendor handles infrastructure and you pay per GB. The trade-off is less control over where data lives and dependency on the vendor. Many organizations now use hybrid - cloud SIEM with on-prem collectors for data that can't leave the network."
