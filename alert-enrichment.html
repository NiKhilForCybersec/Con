Asset Enrichment

SPL: Asset Context Lookup
index=security_alerts
| lookup asset_inventory ip as src_ip 
    OUTPUT hostname as src_hostname, 
           asset_type as src_type,
           owner as src_owner,
           department as src_dept,
           criticality as src_criticality
| lookup asset_inventory ip as dest_ip 
    OUTPUT hostname as dest_hostname,
           asset_type as dest_type,
           owner as dest_owner,
           criticality as dest_criticality
| eval priority = case(
    dest_criticality="Critical", "High",
    dest_criticality="High", "Medium",
    true(), severity
)
| table _time, alert_name, src_ip, src_hostname, src_owner,
        dest_ip, dest_hostname, dest_criticality, priority
KQL: Asset Enrichment
SecurityAlert
| where TimeGenerated > ago(24h)
| extend SourceIP = tostring(parse_json(ExtendedProperties).["Source IP"])
| extend DestIP = tostring(parse_json(ExtendedProperties).["Destination IP"])
| join kind=leftouter (
    _GetWatchlist('AssetInventory')
    | project IPAddress, Hostname, AssetType, Owner, Criticality
) on $left.DestIP == $right.IPAddress
| extend EnrichedPriority = case(
    Criticality == "Critical", "High",
    Criticality == "High", "Medium",
    AlertSeverity
)
| project TimeGenerated, AlertName, SourceIP, DestIP,
          Hostname, AssetType, Owner, Criticality, EnrichedPriority

Automated Priority Adjustment

SPL: Dynamic Priority Scoring
index=security_alerts
// Base score from severity
| eval base_score = case(
    severity="critical", 100,
    severity="high", 75,
    severity="medium", 50,
    severity="low", 25,
    0
)
// Asset criticality modifier
| lookup asset_inventory ip as dest_ip OUTPUT criticality
| eval asset_modifier = case(
    criticality="Critical", 1.5,
    criticality="High", 1.2,
    1.0
)
// Threat intel modifier
| lookup threat_intel_ip ip as src_ip OUTPUT threat_type
| eval ti_modifier = if(isnotnull(threat_type), 1.3, 1.0)

// User privilege modifier
| lookup user_directory user OUTPUT privileged
| eval user_modifier = if(privileged="true", 1.4, 1.0)

// Calculate final score
| eval final_score = round(base_score * asset_modifier * ti_modifier * user_modifier)
| eval adjusted_priority = case(
    final_score >= 150, "Critical",
    final_score >= 100, "High",
    final_score >= 50, "Medium",
    "Low"
)
| table _time, alert_name, severity, final_score, adjusted_priority,
        criticality, threat_type, privileged

Q: "How do you implement alert enrichment in a SIEM?"

A: "Enrichment happens at multiple stages:

  1. Ingestion-time enrichment: For static lookups like GeoIP and asset inventory, I enrich during parsing. This adds fields to every event, making them available for any search.
  2. Search-time enrichment: For dynamic data or expensive lookups, I use search-time joins. Threat intel that changes frequently stays in lookup tables joined at query time.
  3. Priority automation: I calculate risk scores combining multiple enrichment factors. Asset criticality Ã— threat intel match Ã— user privilege = dynamic priority that auto-elevates important alerts.
  4. Context panels: In the alert interface, I show enrichment data in context panelsâ€”prior alerts for this IP, user's recent activity, asset details. This accelerates triage.
  5. Freshness management: Stale enrichment data is dangerous. I track lookup table freshness and alert when asset inventory is over 24 hours old or TI feeds haven't updated.

The goal is that when an analyst opens an alert, they have everything needed to make a decision without running additional queries. Good enrichment cuts investigation time significantly."

Key Enrichment Fields

  • Asset: hostname, owner, criticality
  • User: department, privileged, risk score
  • Geo: country, city, ASN
  • TI: threat type, actor, confidence
  • History: prior alerts, first seen

Priority Multipliers

  • Critical asset: 1.5x
  • TI match: 1.3x
  • Privileged user: 1.4x
  • First time seen: 1.2x

Lookup Table Freshness

  • Asset inventory: < 24 hours
  • Threat intel: < 4 hours
  • User directory: < 24 hours
  • GeoIP: < 7 days
