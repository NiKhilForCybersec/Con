Cloud Logs

AWS, Azure, and GCP security logging

Cloud Logging Overview

Each major cloud provider has its own logging ecosystem. Understanding these is essential for cloud security monitoring.

AWS Logging

Service What It Logs SIEM Priority
CloudTrail API calls (management events) Critical
GuardDuty Threat detection findings Critical
VPC Flow Logs Network traffic metadata Important
S3 Access Logs Bucket access Selective

Azure Logging

Log Type What It Contains Sentinel Table
Activity Log Subscription-level operations AzureActivity
Sign-in Logs Azure AD authentications SigninLogs
Diagnostic Logs Resource-specific events Various per resource
Defender for Cloud Security recommendations, alerts SecurityAlert

GCP Logging

  • Cloud Audit Logs: Admin Activity, Data Access, System Event
  • VPC Flow Logs: Network traffic metadata
  • Security Command Center: Findings and recommendations
üí° Cloud Logging Best Practices
  • Enable CloudTrail/Activity Logs in ALL regions (attackers use obscure regions)
  • Centralize logs to dedicated logging account/subscription
  • Enable data event logging selectively (high volume, high cost)
  • Use cloud-native threat detection (GuardDuty, Defender for Cloud) as a signal
üíº Interview Tip

Question: "What would you monitor in AWS CloudTrail?"

Answer: "I focus on high-risk API calls: IAM changes (CreateUser, AttachUserPolicy, CreateAccessKey), security group modifications, S3 bucket policy changes, console logins especially with MFA disabled, root account usage, and AssumeRole events across accounts. I also watch for reconnaissance APIs like DescribeInstances from unusual principals. GuardDuty findings integrate directly with our SIEM for threat detection."
