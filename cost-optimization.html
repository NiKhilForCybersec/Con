// Estimate monthly costs
let PricePerGB = 2.76;  // Example Sentinel price
Usage
| where TimeGenerated > ago(7d)
| summarize DailyGB = sum(Quantity) / 1000 / 7 by DataType
| extend MonthlyGB = DailyGB * 30
| extend MonthlyCost = MonthlyGB * PricePerGB

    TotalMonthlyCost = sum(MonthlyCost)

Q: "How do you manage SIEM costs?"

A: "Cost management starts with understanding value:

  1. Audit current ingestion: I analyze what's being collected and its security value. Often 30-40% of data provides no detection value.
  2. Tiered architecture: Not all logs need real-time SIEM. High-value security events go to SIEM; verbose logs go to cheaper data lakes; noise gets dropped.
  3. Filter at source: Drop debug logs, health checks, and known-good events before they reach the SIEM. Filtering after ingestion still costs.
  4. Commitment planning: For predictable workloads, commitment tiers save 30-50%. I analyze 90 days of usage before committing.
  5. Continuous review: Data volumes creep up. I run monthly ingestion reports and investigate spikes. New applications often over-log initially.
  6. ROI framing: I justify SIEM cost against breach cost and analyst efficiency. A $500K/year SIEM is cheap compared to a $4M breach.

The goal is maximizing security value per dollar, not minimizing spend."
