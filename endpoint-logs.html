EDR telemetry and endpoint security logging

Endpoint Security Logs

EDR (Endpoint Detection and Response) tools provide the richest telemetry for detecting attacks at the execution level.

EDR vs Traditional AV Logs

Traditional AV Modern EDR
Signature-based detection Behavioral + signature detection
Malware detected/blocked Full process tree, command lines
Limited context Network connections, file writes, registry
Reactive alerts only Proactive hunting capability

Key EDR Telemetry

Telemetry Type Detection Use
Process Creation Malware execution, living-off-the-land
Network Connections C2 communication, lateral movement
File Operations Ransomware, data staging
Registry Changes Persistence mechanisms
Script Execution PowerShell, WMI, VBS attacks
Module Loads DLL injection, LSASS access

Major EDR Platforms

  • CrowdStrike Falcon: Market leader, cloud-native
  • Microsoft Defender for Endpoint: Integrated with M365
  • SentinelOne: AI-focused, autonomous response
  • Carbon Black: VMware ecosystem
  • Cortex XDR: Palo Alto Networks platform
üí° SIEM + EDR Integration

Most organizations send EDR alerts to SIEM but NOT raw telemetry (too high volume). Use EDR's native console for hunting, SIEM for correlation with other sources and unified incident management.

üíº Interview Tip

Question: "What's the relationship between EDR and SIEM?"

Answer: "They're complementary. EDR provides deep endpoint visibility and response capabilities - process trees, memory analysis, isolation. SIEM provides cross-source correlation and compliance. Typically, we send EDR alerts and high-fidelity events to SIEM for unified incident management, but keep detailed hunting in EDR's native interface. Some organizations also forward select telemetry like process creation to enable SIEM-based hunting across endpoint and other sources."
