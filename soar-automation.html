Automation Levels

Not all automation is equal. Start simple and build up:

Level Description Examples Risk
Level 1: Enrichment Add context to alerts automatically IP reputation, user info, asset data, related alerts Very Low
Level 2: Notification Alert the right people Slack/Teams alerts, email to on-call, ticket creation Low
Level 3: Triage Auto-classify obvious cases Close known false positives, set priority Low-Medium
Level 4: Investigation Gather investigation data Query SIEM for related events, check threat intel Medium
Level 5: Containment Take response actions Block IP, disable account, isolate host High

Start with Level 1-2 automation. Only automate containment (Level 5) for high-confidence, high-impact scenarios. Always include human approval for destructive actions until confidence is established.

High-Value Automations

1. Alert Enrichment

Add context to every alert automatically:

  • User context: Department, manager, VIP status, recent activity
  • Asset context: Criticality, owner, OS, patch status
  • Threat intel: IP/domain/hash reputation
  • Historical context: Similar alerts, false positive history

2. Automated Triage

Close or deprioritize obvious cases:

  • Known false positive patterns (same user + alert + action)
  • Alerts from authorized security tools
  • Scheduled task activity during maintenance windows
  • Test/dev environment alerts (deprioritize)

3. Investigation Assistance

Gather data analysts would manually collect:

  • Related events from SIEM (timeline queries)
  • User's recent activity
  • Network connections from the host
  • Process execution history

4. Containment Actions

For high-confidence threats only:

  • Account: Disable, force password reset, revoke sessions
  • Network: Block IP in firewall, isolate host
  • Endpoint: Quarantine file, isolate from network
  • Email: Delete message from mailboxes

Automation Metrics

Metric Description Target
Automation Rate % of alerts with automated enrichment >90%
Auto-Close Rate % of alerts closed automatically 20-40%
Time Saved Analyst hours saved per week Track trend
Playbook Success Rate % of playbook runs completing successfully >95%
False Auto-Close Rate Auto-closed alerts that were actually true positives <1%

Automation Best Practices

Design Principles

  • Start simple: Enrich â†’ Notify â†’ Triage â†’ Investigate â†’ Contain
  • Human in the loop: Require approval for destructive actions initially
  • Fail safe: If automation fails, alert should go to analyst, not be dropped
  • Audit everything: Log all automated actions for review
  • Test thoroughly: Test playbooks with real and synthetic data

Avoid These Mistakes

  • âŒ Automating containment without high confidence
  • âŒ Complex playbooks that are hard to debug
  • âŒ No error handling in playbooks
  • âŒ Automating before understanding the manual process
  • âŒ No monitoring of automation health
