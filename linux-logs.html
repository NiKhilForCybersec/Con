Linux Logs

Linux logging for security monitoring

Linux Logging Landscape

Linux logging is more fragmented than Windows, with multiple systems depending on distribution and version:

System What It Does Location
syslog/rsyslog Traditional system logging /var/log/syslog, /var/log/messages
journald Systemd's logging (modern distros) journalctl command, binary format
auditd Linux Audit Framework (security) /var/log/audit/audit.log
Application logs Individual application logging /var/log/apache2/, /var/log/nginx/, etc.

Critical Log Files

Log File Contents Security Value
/var/log/auth.log Authentication events (Debian/Ubuntu) Critical
/var/log/secure Authentication events (RHEL/CentOS) Critical
/var/log/audit/audit.log Audit framework events Critical
/var/log/cron Scheduled task execution Important
/var/log/sudo.log Sudo command usage Critical

Linux Audit Framework (auditd)

Auditd provides detailed security auditing similar to Windows Security events:

üí° Key Audit Rules

Configure these for security monitoring:

  • -w /etc/passwd -p wa - Watch password file changes
  • -w /etc/shadow -p wa - Watch shadow file changes
  • -w /etc/sudoers -p wa - Watch sudo config changes
  • -a always,exit -F arch=b64 -S execve - Log all command execution

Collection Methods

Method How It Works Best For
Syslog forwarding rsyslog sends to remote syslog receiver Simple setup, all distros
Splunk UF Monitor log files directly Splunk deployments
Azure Monitor Agent Syslog + custom logs to Sentinel Azure/Sentinel deployments
Filebeat Lightweight shipper Elastic deployments
üíº Interview Tip

Question: "How would you detect privilege escalation on Linux?"

Answer: "I'd focus on several indicators: sudo usage from auth.log or sudo.log, looking for unusual users or commands. Auditd logs for setuid/setgid changes and execve calls, especially for known escalation tools like pkexec. Watch for /etc/passwd and /etc/shadow modifications. Monitor cron for persistence. And process execution logs for common exploit patterns - unexpected root processes, shells spawning from web servers, etc."
