Network Logs

Firewall, proxy, and network device logging

Network Log Sources

Network logs provide visibility into traffic patterns, policy violations, and potential threats at the network layer:

Source Type What It Shows Detection Value
Firewall Allow/deny decisions, traffic flow Policy violations, lateral movement, C2
Web Proxy HTTP/HTTPS requests, URLs, user agent Malware callbacks, data exfiltration
DNS Domain queries and responses C2, DGA, DNS tunneling
IDS/IPS Signature-based threat detection Known attack patterns, exploits
NetFlow/IPFIX Traffic metadata (not content) Anomaly detection, traffic patterns
VPN Remote access sessions Compromised credentials, impossible travel

Firewall Log Essentials

Key fields to collect from firewalls:

  • Action: Allow, deny, drop, reset
  • Source/Dest IP: Traffic endpoints
  • Source/Dest Port: Service identification
  • Protocol: TCP, UDP, ICMP
  • Bytes/Packets: Volume (exfiltration detection)
  • Rule/Policy: Which rule matched
  • Zone: Inside, outside, DMZ

DNS Logging for Security

üí° Why DNS is Gold for Detection

Almost all malware needs DNS to function. Attackers can encrypt traffic, but they still need to resolve C2 domains. DNS logging enables:

  • C2 beacon detection (periodic queries to unusual domains)
  • DGA detection (algorithmically generated domains)
  • DNS tunneling (data hidden in DNS queries)
  • Brand impersonation (typosquatting)

Collection Considerations

Challenge Solution
High volume Filter allowed traffic, focus on denies and anomalies
Various formats CEF/LEEF normalization, vendor-specific parsers
TLS inspection gaps Proxy with SSL inspection, or focus on metadata
Syslog reliability Use TCP syslog, implement buffering
üíº Interview Tip

Question: "How would you detect data exfiltration using network logs?"

Answer: "I'd look for several indicators: unusual outbound data volumes to uncommon destinations in firewall logs, DNS queries for large TXT records or high query volumes to single domains (DNS tunneling), proxy logs showing uploads to file-sharing sites or cloud storage, connections to known-bad IPs from threat intelligence, and after-hours traffic from sensitive systems. Baseline normal traffic patterns first, then alert on deviations."
