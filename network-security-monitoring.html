C2 Detection Techniques

Beaconing Detection

Malware often "beacons" home at regular intervals. Detecting this pattern:

SPL: Beaconing Detection
index=firewall action=allowed dest_port IN (80, 443)
| bucket _time span=1m
| stats count as connections by src_ip, dest_ip, dest_port, _time
| streamstats window=60 stdev(connections) as conn_stdev, 
              avg(connections) as conn_avg by src_ip, dest_ip
| where conn_stdev < 0.5 AND conn_avg > 0  // Very regular pattern
| stats count as beacon_minutes, 
        values(dest_port) as ports,
        avg(conn_avg) as avg_conns_per_min
  by src_ip, dest_ip
| where beacon_minutes > 30  // Sustained for 30+ minutes
| sort -beacon_minutes
KQL: Beaconing Detection
CommonSecurityLog
| where TimeGenerated > ago(4h)
| where DeviceAction == "allow"
| where DestinationPort in (80, 443)
| summarize ConnectionCount = count() by SourceIP, DestinationIP, 
            bin(TimeGenerated, 1m)
| summarize 
    TotalMinutes = count(),
    StdDev = stdev(ConnectionCount),
    AvgConn = avg(ConnectionCount)
  by SourceIP, DestinationIP
| where StdDev < 0.5 and TotalMinutes > 30
| order by TotalMinutes desc

Unusual Port Usage

SPL: Non-Standard Ports
index=firewall action=allowed 
| where NOT dest_port IN (80, 443, 53, 25, 22, 21, 3389, 445, 135, 389, 636)
| where dest_port > 1024
| stats count, dc(dest_ip) as unique_dests, values(dest_ip) as sample_dests 
  by src_ip, dest_port
| where count > 100 OR unique_dests > 10
| sort -count

Long Connection Detection

SPL: Long-Lived Connections
index=firewall action=allowed
| where duration > 3600  // Connections over 1 hour
| where NOT dest_port IN (443, 22)  // Exclude expected long connections
| stats count, sum(bytes_out) as total_bytes, max(duration) as max_duration 
  by src_ip, dest_ip, dest_port
| eval duration_hours = round(max_duration/3600, 1)
| where total_bytes > 10000000  // Over 10MB transferred
| sort -total_bytes

SPL: Workstation-to-Workstation SMB
index=firewall dest_port=445 action=allowed
| lookup asset_inventory ip as src_ip OUTPUT asset_type as src_type
| lookup asset_inventory ip as dest_ip OUTPUT asset_type as dest_type
| where src_type="workstation" AND dest_type="workstation"
| stats count, dc(dest_ip) as targets, values(dest_ip) as target_ips 
  by src_ip
| where targets > 3  // Scanning multiple workstations
| sort -targets
KQL: Unusual RDP Sources
// RDP from workstations (not jump servers)
CommonSecurityLog
| where DestinationPort == 3389 and DeviceAction == "allow"
| join kind=leftouter (
    _GetWatchlist('JumpServers') | project SourceIP = IPAddress
) on SourceIP
| where isempty(SourceIP1)  // Not from approved jump server
| summarize 
    ConnectionCount = count(),
    UniqueDestinations = dcount(DestinationIP),
    Destinations = make_set(DestinationIP, 10)
  by SourceIP
| where ConnectionCount > 5 or UniqueDestinations > 2

Data Exfiltration Detection

SPL: Large Outbound Transfers
index=proxy action=allowed 
| where bytes_out > 50000000  // Over 50MB
| lookup threat_intel domain as dest_domain OUTPUT threat_category
| stats sum(bytes_out) as total_bytes, 
        count as request_count,
        values(url_category) as categories,
        values(threat_category) as threat_info
  by src_ip, dest_domain, user
| eval total_mb = round(total_bytes/1024/1024, 2)
| where total_mb > 100
| sort -total_mb
SPL: Upload to Cloud Storage
index=proxy http_method IN (POST, PUT)
| where url_category IN ("Cloud Storage", "File Sharing", "Personal Storage")
| where bytes_out > 10000000
| stats sum(bytes_out) as total_uploaded, 
        count as upload_count,
        values(dest_domain) as destinations
  by src_ip, user
| eval uploaded_mb = round(total_uploaded/1024/1024, 2)
| where uploaded_mb > 50
| sort -uploaded_mb

Network Baseline Anomalies

First-Time Connections

SPL: New External Destination
index=firewall action=allowed direction=outbound
| where NOT [search index=firewall earliest=-30d latest=-1d 
             | stats count by dest_ip | fields dest_ip]
| stats count, values(dest_port) as ports, values(src_ip) as sources 
  by dest_ip
| lookup geoip ip as dest_ip OUTPUT country, city
| sort -count

Traffic Volume Anomalies

SPL: Volume Spike Detection
index=firewall action=allowed
| bucket _time span=1h
| stats sum(bytes) as hourly_bytes by src_ip, _time
| streamstats window=168 avg(hourly_bytes) as baseline_avg, 
                        stdev(hourly_bytes) as baseline_stdev by src_ip
| eval upper_threshold = baseline_avg + (baseline_stdev * 3)
| where hourly_bytes > upper_threshold AND hourly_bytes > 100000000
| eval anomaly_factor = round(hourly_bytes / baseline_avg, 1)
| sort -anomaly_factor

Q: "How do you detect command and control traffic?"

A: "I use multiple detection approaches because C2 varies widely:

  1. Beaconing analysis: Looking for regular, periodic connectionsâ€”malware often calls home at fixed intervals. I measure connection timing variance; legitimate traffic is irregular while beacons are suspiciously consistent.
  2. DNS anomalies: High query volume to single domain, long subdomain strings (DNS tunneling), newly registered domains, or domains with high entropy names.
  3. Protocol anomalies: HTTP traffic on non-standard ports, unusual User-Agent strings, or TLS connections to IPs (not domains).
  4. Threat intelligence: Known C2 infrastructureâ€”domains, IPs, JA3 fingerprints.
  5. Behavioral baselines: Connections to destinations never seen before, unusual data volumes, or connections at odd hours.

I layer these detections because sophisticated attackers vary their C2 to evade single detection methods."

Suspicious Ports (Workstation-to-Workstation)

  • 445 - SMB (PsExec, file shares)
  • 135 - RPC (WMI, DCOM)
  • 3389 - RDP
  • 5985/5986 - WinRM
  • 22 - SSH

C2 Indicators

  • Regular beacon intervals (low variance)
  • Long connection durations
  • Unusual destination countries
  • High DNS query volume
  • Encoded payloads in HTTP

Exfiltration Indicators

  • Large uploads (>100MB)
  • Cloud storage POST/PUT
  • After-hours transfers
  • Encrypted uploads to unknown sites
