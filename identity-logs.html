Identity Logs

Active Directory, Azure AD, and identity provider logging

Identity Logs Overview

Identity is the new perimeter. Identity logs are among the most valuable for detecting account compromise, lateral movement, and privilege escalation.

Key Identity Sources

Source What It Covers Key Events
Active Directory (DC logs) On-prem authentication, group changes 4624, 4625, 4768, 4769, 4728, 4732
Azure AD / Entra ID Cloud identity, SSO, Conditional Access SigninLogs, AuditLogs, RiskyUsers
Okta Third-party IdP authentication, user lifecycle, app access
Duo MFA MFA success/failure, bypass, enrollment

Critical Identity Events

Event Why It Matters
Failed logins Password spray, brute force
MFA bypass/failure Attacker trying to circumvent MFA
Impossible travel Login from two locations too far apart
New device/location Potential credential theft
Privilege elevation User added to admin group
Service account misuse Interactive login from service account
üíº Interview Tip

Question: "How would you detect a compromised account?"

Answer: "Multiple signals: impossible travel alerts (login from NYC then London 30 minutes later), successful login after multiple failures from different IPs (password spray success), MFA prompt flood followed by approval (MFA fatigue attack), login from new device combined with sensitive actions like password reset or forwarding rule creation. I also look for behavioral anomalies - user accessing systems they've never touched, downloading unusual amounts of data, or logging in at unusual times."
