Common Interview Questions

Top 50 SIEM consultant interview questions with answers

SIEM Fundamentals Questions

üíº Q1: What is SIEM and why is it important?

Answer: SIEM (Security Information and Event Management) is a central platform that collects logs from across an organization's IT environment, correlates them to detect threats, and alerts security teams to investigate. It's important because modern attacks span multiple systems - without SIEM, security teams have blind spots and can't connect the dots between related events across different sources.

üíº Q2: What's the difference between SIEM and SOAR?

Answer: SIEM focuses on detection - collecting logs, correlating events, and generating alerts. SOAR (Security Orchestration, Automation, and Response) focuses on response - automating actions, orchestrating workflows, and integrating tools. They're complementary: SIEM detects the threat, SOAR helps respond to it. Many platforms now combine both - Splunk SOAR, Sentinel with Logic Apps.

üíº Q3: How would you reduce false positives?

Answer: Start by understanding why they're firing - is the logic too broad, or is the environment unique? Tuning options include: adding exclusions for known-good behavior, increasing thresholds, adding time-based conditions (business hours only), enriching with context (is this user an admin?), and correlating with other signals. Document every tuning change. For persistent issues, consider replacing signature-based rules with behavioral detection.

Technical Questions

üíº Q4: Explain how you'd detect a brute force attack.

Answer: Look for multiple failed authentication attempts from the same source in a short time window. In Splunk: search failed logins, stats count by source IP, where count > 5 in 5 minutes. Include Logon Type in Windows to distinguish network vs local. Consider excluding service accounts and known scanning tools. Alert should include the source IP, targeted accounts, and success/failure after the attempts.

üíº Q5: What Windows event IDs would you monitor for lateral movement?

Answer: 4624 Type 3 (network logon), 4648 (explicit credentials/runas), 4688 (process creation) for tools like PsExec, 5140/5145 (network share access). Also monitor service creation (7045) and scheduled task creation (4698). Correlate with network logs showing SMB (445) or RDP (3389) between internal hosts.

üíº Q6: How do you size a SIEM deployment?

Answer: Key inputs: daily ingest volume (GB/day), retention requirements, search performance needs, and number of concurrent users. For Splunk on-prem, this translates to indexer count and storage. For cloud platforms, it's about selecting the right tier. I always recommend starting with a pilot to measure actual volume from key sources before committing to production sizing.

Scenario Questions

üíº Q7: A critical detection stopped working. How do you troubleshoot?

Answer: Systematic approach: First, check if the rule is enabled and hasn't been modified. Second, verify data is flowing - run the base query manually. Third, check if source format changed (vendor update, new parser needed). Fourth, validate time windows and thresholds still make sense. Fifth, check for infrastructure issues - indexer health, search head capacity. Document the issue and resolution for future reference.

üíº Q8: How would you approach migrating 500 detection rules?

Answer: Don't migrate all 500 blindly. First, categorize by priority and usage - which rules actually fire and get investigated? Migrate critical/high-fidelity rules first. Some rules may be obsolete or have native equivalents in the new platform. Plan for query translation (SPL to KQL or vice versa) and test each migrated rule. Expect 20-30% to need significant rework due to platform differences.

Platform-Specific Questions

üíº Q9: What's the difference between Splunk's Universal and Heavy Forwarder?

Answer: Universal Forwarder is lightweight (~50MB RAM), only forwards raw data. Heavy Forwarder is a full Splunk instance that can parse, filter, and route data before forwarding. Use UF on endpoints; use HF as an aggregation/parsing layer for syslog or when you need to reduce data before sending to cloud.

üíº Q10: When would you choose Sentinel over Splunk?

Answer: Sentinel for Microsoft-heavy environments (M365, Azure, Defender suite) where native integration provides quick value. Also good for cloud-first organizations that don't want infrastructure management. Splunk for multi-cloud environments, on-prem requirements, or when you need maximum flexibility with diverse log sources. Both are capable - it's about fit for the organization.
